📚 자동화 테스트란?

자동화 테스트의 기본 개념

자동화 테스트는 사람이 반복적으로 수행해야 하는 작업을 프로그램으로 자동으로 수행하게 만드는 것입니다.

💡 쉽게 이해하기

자동화 테스트는 마치 로봇이 웹사이트를 대신 사용하는 것과 같습니다. 예를 들어:

  • 매일 아침 로그인하는 대신 → 프로그램이 자동으로 로그인
  • 100개의 제품을 검색하는 대신 → 프로그램이 자동으로 검색
  • 새로운 기능이 추가될 때마다 수동으로 테스트하는 대신 → 프로그램이 자동으로 테스트

왜 자동화 테스트가 필요할까요?

자동화 테스트를 사용하면 많은 이점이 있습니다. 각각을 자세히 알아봅시다:

✅ 시간 절약

문제: 매일 같은 작업을 100번 반복해야 한다면?

해결: 프로그램이 자동으로 처리하면 사람은 다른 중요한 일에 집중할 수 있습니다.

예시: 100개 제품 검색을 수동으로 하면 2시간, 자동화하면 5분!

✅ 정확성

문제: 사람은 실수를 할 수 있습니다. (오타, 클릭 누락 등)

해결: 프로그램은 정확히 같은 방식으로 작업을 수행합니다.

예시: 로그인 테스트를 100번 해도 실수 없이 정확하게!

✅ 효율성

문제: 여러 브라우저(Chrome, Firefox 등)에서 각각 테스트해야 한다면?

해결: 프로그램이 여러 브라우저에서 동시에 테스트할 수 있습니다.

예시: 5개 브라우저를 동시에 테스트하여 시간을 5분의 1로 단축!

✅ 회귀 테스트

문제: 새로운 기능을 추가했는데 기존 기능이 깨졌는지 확인해야 한다면?

해결: 프로그램이 모든 기존 기능을 자동으로 테스트합니다.

예시: 로그인 기능을 수정했는데, 기존 결제 기능이 여전히 작동하는지 자동 확인!

📖 용어 설명:
  • 회귀 테스트: 새로운 기능 추가나 수정 후 기존 기능이 여전히 정상 작동하는지 확인하는 테스트
  • 자동화: 사람이 직접 하던 일을 프로그램이나 시스템이 자동으로 수행하게 하는 것

🌐 Selenium

Selenium이란?

웹 브라우저를 자동으로 제어할 수 있게 해주는 도구입니다. 마치 로봇 손이 웹 브라우저를 조작하는 것과 같습니다.

Selenium이 할 수 있는 일:

  • 🖱️ 클릭: 버튼, 링크를 자동으로 클릭
  • ⌨️ 입력: 텍스트 박스에 글자를 자동으로 입력
  • 🔍 검색: 웹페이지에서 원하는 정보를 찾기
  • 📄 페이지 이동: 다른 페이지로 자동으로 이동
  • 📸 스크린샷: 화면을 자동으로 저장

예시:

사람: "구글에 접속 → 검색창 클릭 → 'Python' 입력 → Enter 키 누름"

Selenium: 프로그램이 위 과정을 자동으로 수행!

🤝 Python과 Selenium의 관계

비유: Python은 언어이고, Selenium은 도구입니다.

  • Python: 영어 (사람과 대화할 수 있는 언어)
  • Selenium: 리모컨 (TV를 조작할 수 있는 도구)

Python이라는 언어로 Selenium이라는 도구를 사용해서 웹 브라우저를 제어하는 것입니다!

1단계: Python 설치

Python이란? 프로그래밍 언어입니다. 자동화 스크립트를 작성하기 위해 필요합니다.

Python이 이미 설치되어 있는지 확인하기

먼저 Python이 이미 설치되어 있는지 확인해봅시다.

🔍 터미널/명령 프롬프트란?

Windows: "명령 프롬프트" 또는 "PowerShell"이라고 부릅니다. 검색창에 "cmd" 또는 "PowerShell"을 입력하면 나옵니다.

Mac/Linux: "터미널"이라고 부릅니다. 응용 프로그램에서 찾을 수 있습니다.

터미널은 텍스트로 컴퓨터에게 명령을 내리는 곳입니다. 검은 화면에 글자가 보이는 그 화면이에요!

터미널/명령 프롬프트에서 확인 
python --version
# 또는
python3 --version
📝 설명:
  • python --version: Python 버전을 확인하는 명령어입니다
  • 만약 "Python 3.8.0" 같은 메시지가 나오면 → 이미 설치되어 있습니다! ✅
  • 만약 "python은(는) 내부 또는 외부 명령...으로 인식되지 않습니다" 같은 에러가 나오면 → Python을 설치해야 합니다

Python 설치하기 (설치되어 있지 않은 경우)

  1. Python 공식 웹사이트 방문: https://www.python.org/downloads/
  2. 다운로드: 웹사이트에서 자동으로 맞는 버전을 추천해줍니다. 큰 노란색 "Download Python" 버튼을 클릭하세요
  3. 설치 파일 실행: 다운로드한 파일을 더블클릭하여 실행합니다
  4. 중요! 설치 시 "Add Python to PATH" 체크박스를 반드시 체크하세요! (이것을 체크하지 않으면 나중에 문제가 생길 수 있습니다)
  5. 설치 완료: "Install Now" 또는 "설치" 버튼을 클릭하고 완료될 때까지 기다립니다
  6. 확인: 다시 터미널을 열고 python --version을 입력하여 설치가 완료되었는지 확인합니다

⚠️ 설치 시 주의사항

  • "Add Python to PATH" 체크: 이걸 체크하지 않으면 터미널에서 Python을 찾을 수 없습니다
  • 버전: Python 3.8 이상이 필요합니다 (3.9, 3.10, 3.11 등 최신 버전 권장)

2단계: Selenium 설치

Selenium이란? 웹 브라우저를 자동으로 제어할 수 있게 해주는 Python 라이브러리입니다.

pip란?

pip는 Python 패키지 설치 도구입니다. Python을 설치하면 자동으로 함께 설치됩니다.

pip는 마치 앱 스토어처럼, Python으로 만든 프로그램(Selenium 등)을 쉽게 설치할 수 있게 해줍니다.

터미널/명령 프롬프트에서 실행 
pip install selenium
# 또는
pip3 install selenium
📝 설명:
  • pip install selenium: Selenium을 설치하는 명령어입니다
  • Windows에서는 보통 pip를 사용합니다
  • Mac/Linux에서는 pip3를 사용해야 할 수도 있습니다
  • 명령어를 입력하고 Enter 키를 누르면 설치가 시작됩니다
  • 설치가 완료되면 "Successfully installed selenium" 같은 메시지가 나옵니다 ✅

💡 설치 중 나올 수 있는 메시지

  • "Collecting selenium...": Selenium을 다운로드하고 있습니다. 정상입니다!
  • "Successfully installed selenium": 설치 완료! ✅
  • "Requirement already satisfied": 이미 설치되어 있습니다. 괜찮습니다!
  • "pip is not recognized": pip가 없다는 뜻입니다. Python을 다시 설치하거나 PATH를 확인해야 합니다

3단계: 웹드라이버 설치

웹드라이버란? Selenium이 브라우저를 제어하기 위해 필요한 중간 다리 역할을 하는 프로그램입니다.

🎉 좋은 소식!

Selenium 4.x 버전부터는 웹드라이버를 자동으로 다운로드하고 관리합니다!

이전 버전(Selenium 3.x)에서는 수동으로 웹드라이버를 다운로드하고 경로를 설정해야 했지만, 이제는 그럴 필요가 없습니다.

💡 팁: Selenium 4.x는 웹드라이버를 자동으로 관리합니다. 별도 설치가 필요 없습니다!

Chrome 브라우저 확인

Chrome 브라우저가 설치되어 있는지 확인하세요. Chrome이 설치되어 있지 않다면 여기서 다운로드하세요.

Selenium 4.x는 Chrome 브라우저를 자동으로 감지하고 필요한 드라이버를 자동으로 다운로드합니다.

4단계: 첫 번째 테스트 실행

설치가 제대로 되었는지 확인하기 위해 간단한 테스트를 실행해봅시다!

테스트 파일 만들기

  1. 텍스트 에디터 열기: 메모장(Windows) 또는 텍스트 에디트(Mac)를 엽니다
  2. 파일 생성: 아래 코드를 복사해서 붙여넣습니다
  3. 저장: 파일을 test_install.py라는 이름으로 저장합니다 (반드시 .py 확장자로 저장하세요!)
  4. 위치: 저장할 위치는 어디든 상관없지만, 찾기 쉬운 곳(예: 바탕화면)에 저장하는 것을 권장합니다
test_install.py 
from selenium import webdriver
from selenium.webdriver.chrome.service import Service
from selenium.webdriver.chrome.options import Options

# Chrome 옵션 설정
chrome_options = Options()
chrome_options.add_argument('--headless')  # 백그라운드 실행 (선택사항)

# Selenium 4.x 자동 드라이버 관리
driver = webdriver.Chrome(options=chrome_options)

# 구글 홈페이지 열기
driver.get("https://www.google.com")

# 페이지 제목 확인
print(f"페이지 제목: {driver.title}")

# 브라우저 닫기
driver.quit()

print("✅ 설치가 완료되었습니다!")

코드 설명 (한 줄씩 이해하기)

📖 코드 설명:
  1. from selenium import webdriver: Selenium 라이브러리에서 webdriver를 가져옵니다. 웹 브라우저를 제어하기 위해 필요합니다.
  2. from selenium.webdriver.chrome.service import Service: Chrome 서비스를 위한 도구를 가져옵니다 (현재는 사용하지 않지만, 나중에 필요할 수 있습니다).
  3. from selenium.webdriver.chrome.options import Options: Chrome 브라우저의 옵션을 설정하기 위한 도구를 가져옵니다.
  4. chrome_options = Options(): Chrome 옵션 객체를 만듭니다. 브라우저를 어떻게 실행할지 설정할 수 있습니다.
  5. chrome_options.add_argument('--headless'): 브라우저를 백그라운드에서 실행하도록 설정합니다 (화면에 보이지 않음). 이 줄은 선택사항이므로 삭제해도 됩니다. 삭제하면 브라우저 창이 보입니다.
  6. driver = webdriver.Chrome(options=chrome_options): Chrome 브라우저를 엽니다. 이제 driver라는 변수로 브라우저를 제어할 수 있습니다.
  7. driver.get("https://www.google.com"): 구글 홈페이지를 엽니다. 브라우저 주소창에 URL을 입력하는 것과 같습니다.
  8. print(f"페이지 제목: {driver.title}"): 웹페이지의 제목을 화면에 출력합니다. "Google" 같은 제목이 나올 것입니다.
  9. driver.quit(): 브라우저를 닫습니다. 작업이 끝나면 반드시 브라우저를 닫아야 합니다.
  10. print("✅ 설치가 완료되었습니다!"): 성공 메시지를 출력합니다.

테스트 실행하기

    • Windows: cd C:\Users\사용자명\Desktop (바탕화면에 저장한 경우)
    • Mac/Linux: cd ~/Desktop (바탕화면에 저장한 경우)
  3. 실행: python test_install.py 또는 python3 test_install.py를 입력하고 Enter 키를 누릅니다
  4. 결과 확인:
    • 성공하면: "페이지 제목: Google"과 "✅ 설치가 완료되었습니다!" 메시지가 나옵니다
    • 실패하면: 에러 메시지가 나옵니다 (아래 트러블슈팅 참고)

트러블슈팅 (문제 해결)

❌ 문제가 발생했다면?

  • "ModuleNotFoundError: No module named 'selenium'": Selenium이 설치되지 않았습니다. pip install selenium을 다시 실행하세요.
  • "'chromedriver' executable needs to be in PATH": Selenium 4.x를 최신 버전으로 업데이트하세요: pip install --upgrade selenium
  • "Chrome 브라우저를 찾을 수 없습니다": Chrome 브라우저를 설치하세요.
  • "python: command not found": Python이 설치되지 않았거나 PATH에 추가되지 않았습니다. Python을 다시 설치하세요.

1. 기본 사용법

Selenium을 사용하는 모든 코드는 기본적으로 4단계로 구성됩니다. 이 패턴을 이해하면 모든 자동화를 할 수 있습니다!

📋 Selenium 사용의 4단계

  1. 브라우저 열기: 작업할 브라우저를 실행합니다
  2. 웹페이지 열기: 작업할 웹페이지를 엽니다
  3. 요소 찾기 및 상호작용: 웹페이지의 버튼이나 입력창을 찾아서 클릭하거나 입력합니다
  4. 브라우저 닫기: 작업이 끝나면 브라우저를 닫습니다

이 4단계가 모든 Selenium 코드의 기본 구조입니다!

기본 구조 
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.common.keys import Keys
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

# 1. 브라우저 열기
driver = webdriver.Chrome()

# 2. 웹페이지 열기
driver.get("https://example.com")

# 3. 요소 찾기 및 상호작용
element = driver.find_element(By.ID, "element-id")
element.click()

# 4. 브라우저 닫기
driver.quit()

코드 설명 (한 줄씩 이해하기)

📖 각 줄 설명:
  1. from selenium import webdriver: Selenium 라이브러리에서 웹 브라우저를 제어하는 도구(webdriver)를 가져옵니다.
  2. from selenium.webdriver.common.by import By: 요소를 찾는 방법(ID, 클래스명 등)을 사용하기 위해 가져옵니다.
  3. from selenium.webdriver.common.keys import Keys: 키보드 키(Enter, Tab 등)를 사용하기 위해 가져옵니다.
  4. from selenium.webdriver.support.ui import WebDriverWait: 웹페이지가 로드될 때까지 기다리는 기능을 사용하기 위해 가져옵니다.
  5. from selenium.webdriver.support import expected_conditions as EC: 대기 조건(예: 요소가 클릭 가능할 때까지)을 사용하기 위해 가져옵니다.
  6. driver = webdriver.Chrome(): Chrome 브라우저를 엽니다. driver는 브라우저를 제어하는 리모컨과 같습니다.
  7. driver.get("https://example.com"): 지정한 웹사이트 주소로 이동합니다. 브라우저 주소창에 URL을 입력하는 것과 같습니다.
  8. element = driver.find_element(By.ID, "element-id"): 웹페이지에서 ID가 "element-id"인 요소를 찾습니다. element는 찾은 요소를 저장하는 변수입니다.
  9. element.click(): 찾은 요소를 클릭합니다. 마우스로 버튼을 클릭하는 것과 같습니다.
  10. driver.quit(): 브라우저를 완전히 닫습니다. 작업이 끝나면 반드시 이 명령어를 실행해야 합니다!

요소를 찾는 8가지 방법

같은 요소를 찾는 방법이 여러 가지 있습니다. 각 방법의 특징을 알아봅시다:

요소 찾기 방법 
from selenium.webdriver.common.by import By

# ID로 찾기 (가장 빠름)
element = driver.find_element(By.ID, "username")

# 클래스명으로 찾기
element = driver.find_element(By.CLASS_NAME, "button-primary")

# 이름 속성으로 찾기
element = driver.find_element(By.NAME, "email")

# 태그명으로 찾기
element = driver.find_element(By.TAG_NAME, "button")

# CSS 선택자로 찾기 (유연함)
element = driver.find_element(By.CSS_SELECTOR, "#login-form .submit-btn")

# XPath로 찾기 (복잡한 경우)
element = driver.find_element(By.XPATH, "//button[@type='submit']")

# 링크 텍스트로 찾기
element = driver.find_element(By.LINK_TEXT, "로그인")

# 부분 링크 텍스트로 찾기
element = driver.find_element(By.PARTIAL_LINK_TEXT, "로그인")

각 방법의 특징과 사용 시기

1. ID로 찾기

언제 사용? 요소에 고유한 ID가 있을 때

장점: 가장 빠르고 정확함

예시: driver.find_element(By.ID, "username")

비유: 집 주소처럼 정확한 위치를 알려주는 것

2. Name으로 찾기

언제 사용? 폼 요소(input, select 등)를 찾을 때

장점: 폼에서 자주 사용되는 방법

예시: driver.find_element(By.NAME, "email")

비유: 이름표로 사람을 찾는 것

3. CSS 선택자

언제 사용? 복잡한 조건으로 찾아야 할 때

장점: 매우 유연하고 강력함

예시: driver.find_element(By.CSS_SELECTOR, "#login-form .submit-btn")

비유: 여러 조건을 조합해서 찾는 것

4. XPath

언제 사용? 다른 방법으로 찾을 수 없을 때

장점: 가장 강력함 (부모-자식 관계 등)

단점: 느릴 수 있고 복잡함

예시: driver.find_element(By.XPATH, "//button[@type='submit']")

3. 주요 동작들

요소를 찾았으면 이제 그것을 조작해야 합니다. 웹페이지에서 가장 자주 사용하는 동작들을 알아봅시다.

🎯 자동화의 핵심 동작

자동화는 기본적으로 다음 4가지 동작의 조합입니다:

  1. 입력: 텍스트를 입력합니다
  2. 클릭: 버튼이나 링크를 클릭합니다
  3. 읽기: 화면의 텍스트나 정보를 읽습니다
  4. 이동: 다른 페이지로 이동합니다
주요 동작 예제 
from selenium.webdriver.common.by import By
from selenium.webdriver.common.keys import Keys

# 텍스트 입력
input_field = driver.find_element(By.ID, "search")
input_field.send_keys("자동화 테스트")
input_field.clear()  # 입력 필드 지우기

# 클릭
button = driver.find_element(By.ID, "submit-btn")
button.click()

# 키보드 입력
input_field.send_keys(Keys.ENTER)  # Enter 키
input_field.send_keys(Keys.TAB)    # Tab 키
input_field.send_keys(Keys.ESCAPE) # ESC 키

# 텍스트 가져오기
text = element.text
print(f"요소의 텍스트: {text}")

# 속성 가져오기
value = element.get_attribute("href")
print(f"링크 주소: {value}")

# 페이지 이동
driver.get("https://example.com")
driver.back()    # 뒤로 가기
driver.forward() # 앞으로 가기
driver.refresh() # 새로고침

# 현재 URL 가져오기
current_url = driver.current_url
print(f"현재 주소: {current_url}")

# 페이지 제목 가져오기
title = driver.title
print(f"페이지 제목: {title}")

각 동작 상세 설명

1. 텍스트 입력

send_keys("텍스트"): 입력창에 텍스트를 입력합니다

예시: input_field.send_keys("자동화 테스트")

비유: 키보드로 타이핑하는 것과 같습니다

clear(): 입력창의 내용을 지웁니다

예시: input_field.clear()

2. 클릭

click(): 요소를 클릭합니다

예시: button.click()

비유: 마우스로 버튼을 클릭하는 것과 같습니다

주의: 요소가 클릭 가능한 상태여야 합니다

3. 키보드 입력

Keys.ENTER: Enter 키를 누릅니다

Keys.TAB: Tab 키를 누릅니다

Keys.ESCAPE: ESC 키를 누릅니다

예시: input_field.send_keys(Keys.ENTER)

비유: 키보드의 특수 키를 누르는 것과 같습니다

4. 텍스트 읽기

.text: 요소에 표시된 텍스트를 가져옵니다

예시: text = element.text

비유: 화면에 보이는 글자를 읽는 것과 같습니다

5. 속성 읽기

get_attribute("속성명"): 요소의 속성 값을 가져옵니다

예시: link = element.get_attribute("href")

비유: 링크의 주소, 이미지의 경로 등을 읽는 것

6. 페이지 이동

driver.get("URL"): 새 페이지로 이동

driver.back(): 이전 페이지로 (뒤로 가기)

driver.forward(): 다음 페이지로 (앞으로 가기)

driver.refresh(): 현재 페이지 새로고침

4. 대기 (Wait)

웹페이지는 로드되는 데 시간이 걸립니다. 요소를 찾기 전에 페이지가 완전히 로드될 때까지 기다려야 합니다.

⏰ 왜 대기가 필요한가요?

문제 상황:

  • 페이지를 열었는데 요소가 아직 나타나지 않았습니다
  • 버튼을 클릭하려고 했는데 아직 로딩 중입니다
  • 입력창에 텍스트를 입력하려고 했는데 아직 생성되지 않았습니다

해결: 요소가 나타날 때까지 기다려야 합니다!

대기의 3가지 방법

대기하는 방법은 크게 3가지가 있습니다. 각각의 특징을 알아봅시다:

대기 방법 
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from selenium.webdriver.common.by import By
import time

# 1. 암시적 대기 (Implicit Wait) - 모든 요소에 적용
driver.implicitly_wait(10)  # 최대 10초까지 대기

# 2. 명시적 대기 (Explicit Wait) - 특정 조건까지 대기 (권장)
wait = WebDriverWait(driver, 10)
element = wait.until(EC.presence_of_element_located((By.ID, "my-element")))

# 3. 시간 대기 (명시적 대기 사용 권장)
time.sleep(2)  # 2초 대기 (가능한 피하기)

# 자주 사용하는 대기 조건들
wait.until(EC.element_to_be_clickable((By.ID, "button")))
wait.until(EC.visibility_of_element_located((By.ID, "content")))
wait.until(EC.text_to_be_present_in_element((By.ID, "status"), "완료"))

각 대기 방법 상세 설명

1. 암시적 대기 (Implicit Wait)

설명: 모든 요소를 찾을 때 자동으로 기다립니다

사용법: driver.implicitly_wait(10)

의미: 요소를 찾을 때 최대 10초까지 기다립니다

장점: 간단하고 편리함

단점: 모든 요소에 적용되어 불필요한 대기 시간이 생길 수 있음

비유: 모든 약속에 10분 일찍 도착하는 것

2. 명시적 대기 (Explicit Wait) ⭐ 권장

설명: 특정 조건이 만족될 때까지 기다립니다

사용법: wait.until(EC.presence_of_element_located(...))

의미: 요소가 나타날 때까지 최대 10초까지 기다립니다

장점: 정확하고 효율적 (조건이 만족되면 바로 진행)

단점: 코드가 조금 더 복잡함

비유: 친구가 올 때까지 기다리는 것 (와면 바로 출발)

3. 시간 대기 (time.sleep)

설명: 무조건 지정한 시간만큼 기다립니다

사용법: time.sleep(2)

의미: 무조건 2초 기다립니다

장점: 매우 간단함

단점: 비효율적 (필요 없어도 기다림), 테스트가 느려짐

비유: 무조건 10분 기다리는 것 (빨리 와도 기다림)

⚠️ 권장하지 않음: 가능하면 명시적 대기를 사용하세요!

자주 사용하는 대기 조건

📖 대기 조건 설명:
  • EC.presence_of_element_located: 요소가 HTML에 존재할 때까지 기다립니다 (화면에 보이지 않아도 됨)
  • EC.visibility_of_element_located: 요소가 화면에 보일 때까지 기다립니다 (더 정확함)
  • EC.element_to_be_clickable: 요소가 클릭 가능할 때까지 기다립니다 (버튼 클릭 전에 사용)
  • EC.text_to_be_present_in_element: 요소에 특정 텍스트가 나타날 때까지 기다립니다

💡 대기 사용 팁

  1. 명시적 대기 사용 권장: 가장 효율적이고 정확합니다
  2. 적절한 시간 설정: 너무 짧으면 실패하고, 너무 길면 느려집니다 (보통 10초)
  3. time.sleep 피하기: 가능하면 명시적 대기를 사용하세요
  4. 조건 선택: 클릭할 요소면 element_to_be_clickable, 읽을 요소면 visibility_of_element_located 사용

4. 부모-자식 관계 이해하기

DOM에서 부모-자식 관계를 이해하면 정확한 경로를 만들 수 있습니다:

부모-자식 관계 
# HTML 구조:
<div id="container">              <!-- 부모 -->
  <nav class="tabs">              <!-- 자식 (container의), 부모 (button들의) -->
    <button id="btn1">버튼1</button>  <!-- 자식 (nav의) -->
    <button id="btn2">버튼2</button>  <!-- 자식 (nav의) -->
    <button id="btn3">버튼3</button>  <!-- 자식 (nav의) -->
    <button id="btn4">버튼4</button>  <!-- 자식 (nav의) -->
  </nav>
</div>

# 직접 자식 찾기 (/) - 한 단계 아래만
//div[@id='container']/nav          # container의 직접 자식 nav
//nav/button[4]                     # nav의 직접 자식 네 번째 button

# 모든 후손 찾기 (//) - 모든 단계 아래
//div[@id='container']//button      # container 아래 모든 button (어디에 있든)
//nav//button                       # nav 아래 모든 button

# 부모 찾기
//button[@id='btn4']/parent::nav    # btn4의 부모 nav
//button[@id='btn4']/..             # btn4의 부모 (간단한 방법)

# 조상 찾기
//button[@id='btn4']/ancestor::div  # btn4의 조상 div
//button[@id='btn4']/ancestor::div[@id='container']  # 특정 조상

6. 실제 예제: /html/body/div/nav/button[4] 분석

제공하신 경로를 분석해봅시다:

절대 경로 분석 
# 경로: /html/body/div/nav/button[4]

# 경로 분석:
/html              → html 태그 (루트)
  /body            → body 태그 (html의 자식)
    /div            → div 태그 (body의 자식, 첫 번째 div)
      /nav          → nav 태그 (div의 자식)
        /button[4]  → nav의 네 번째 button 요소

# HTML 구조 예상:
<html>
  <body>
    <div>
      <nav>
        <button>1번</button>
        <button>2번</button>
        <button>3번</button>
        <button>4번</button>  ← 이 요소를 찾음
      </nav>
    </div>
  </body>
</html>

# Selenium에서 사용:
from selenium.webdriver.common.by import By

# 절대 경로 사용
element = driver.find_element(By.XPATH, "/html/body/div/nav/button[4]")

# 더 안정적인 상대 경로 (권장)
element = driver.find_element(By.XPATH, "//nav/button[4]")

# 또는 더 구체적으로
element = driver.find_element(By.XPATH, "//body//nav//button[4]")

# CSS 선택자 사용 (권장)
element = driver.find_element(By.CSS_SELECTOR, "nav button:nth-child(4)")

# 인덱스로 접근
buttons = driver.find_elements(By.CSS_SELECTOR, "nav button")
element = buttons[3]  # 0부터 시작하므로 3번째 = 4번째 요소

/html/body/div[1]/div[1]/nav/button[4]

element = driver.find_element(By.XPATH, "/html/body/div[1]/div[1]/nav/button[4]")

element = driver.find_element(By.CSS_SELECTOR, "nav.tabs button:nth-child(4)")

⚠️ 절대 경로 사용 시 주의사항

  • 절대 경로는 피하세요: HTML 구조가 조금만 바뀌어도 깨질 수 있습니다
  • 상대 경로 권장: `//nav/button[4]` 같은 상대 경로가 더 안정적입니다
  • CSS 선택자 추천: CSS 선택자가 XPath보다 빠르고 읽기 쉽습니다
  • 인덱스 주의: 요소 순서가 바뀌면 인덱스도 바뀝니다
  • 가능하면 ID/Name 사용: 가장 안정적인 방법은 고유한 ID나 Name을 사용하는 것입니다

요소 찾기 전략

요소를 찾는 방법은 여러 가지가 있습니다. 각 방법의 특징과 언제 사용해야 하는지 알아봅시다.

💡 우선순위: ID > Name > CSS Selector > XPath > 기타

✅ 요소 찾기 체크리스트

  1. ID가 있는가? → ID 사용 (가장 빠르고 안정적)
  2. Name 속성이 있는가? → Name 사용 (폼 요소에 적합)
  3. 고유한 클래스가 있는가? → CSS 선택자 사용
  4. 다른 속성이 있는가? → data-*, aria-* 등 활용
  5. 텍스트로만 찾을 수 있는가? → XPath 또는 Link Text 사용
  6. 복잡한 관계인가? → XPath 사용 (부모-자식 관계)
  7. 인덱스가 필요한가? → find_elements 후 인덱스 접근 또는 nth-child 사용

1. ID로 찾기 (가장 권장)

ID는 고유하고 안정적이므로 가장 빠르고 신뢰할 수 있습니다.

ID로 찾기 
from selenium.webdriver.common.by import By

# 단일 요소 찾기
element = driver.find_element(By.ID, "username")
element.click()

# 여러 요소 찾기 (ID는 고유하므로 일반적으로 사용 안 함)
# elements = driver.find_elements(By.ID, "username")  # 리스트 반환

2. Name 속성으로 찾기

폼 요소에 자주 사용되는 name 속성으로 찾습니다.

Name으로 찾기 
from selenium.webdriver.common.by import By

# name="email"인 요소 찾기
email_field = driver.find_element(By.NAME, "email")
email_field.send_keys("test@example.com")

# 여러 요소 찾기
all_inputs = driver.find_elements(By.NAME, "field_name")

3. CSS 선택자로 찾기 (강력함)

CSS 선택자는 매우 유연하고 강력합니다. 복잡한 조건도 표현할 수 있습니다.

CSS 선택자 예제

# 방법 2: find_elements 사용 (예외 없음)
elements = driver.find_elements(By.ID, "my-element")
if elements:
    elements[0].click()
    print("요소를 찾았습니다!")
else:
    print("요소를 찾을 수 없습니다.")

# 방법 3: 명시적 대기 사용 (권장)
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

try:
    element = WebDriverWait(driver, 10).until(
        EC.presence_of_element_located((By.ID, "my-element"))

    element.click()
except:
    print("요소를 찾을 수 없습니다.")

예제 1: 구글 검색 자동화

목표: 구글에서 검색어를 입력하고 결과를 확인하는 자동화를 만들어봅시다.

이 예제는 가장 기본적이고 자주 사용하는 패턴입니다. 이 예제를 이해하면 다른 자동화도 쉽게 만들 수 있습니다!

작업 흐름

🔄 구글 검색 자동화의 단계

  1. 브라우저 열기
  2. 구글 홈페이지 접속
  3. 검색창 찾기 (대기)
  4. 검색어 입력
  5. Enter 키 누르기
  6. 검색 결과 페이지 로드 대기
  7. 검색 결과 제목 가져오기
  8. 브라우저 닫기
google_search.py 
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.common.keys import Keys
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

# 브라우저 열기
driver = webdriver.Chrome()

try:
    # 구글 홈페이지 열기
    driver.get("https://www.google.com")
    
    # 검색창 찾기 (구글의 검색창은 name 속성이 'q')
    search_box = WebDriverWait(driver, 10).until(
        EC.presence_of_element_located((By.NAME, "q"))
    )
    
    # 검색어 입력
    search_box.send_keys("Python Selenium 자동화")
    
    # Enter 키 누르기
    search_box.send_keys(Keys.RETURN)
    
    # 검색 결과 페이지 로드 대기
    WebDriverWait(driver, 10).until(
        EC.presence_of_element_located((By.ID, "search"))
    )
    
    # 검색 결과 제목 가져오기
    results = driver.find_elements(By.CSS_SELECTOR, "h3")
    print(f"\n검색 결과 {len(results)}개를 찾았습니다:\n")
    
    for i, result in enumerate(results[:5], 1):
        print(f"{i}. {result.text}")
    
finally:
    # 브라우저 닫기
    driver.quit()

코드 상세 설명

📖 각 부분 설명:
  1. from selenium import webdriver: Selenium 라이브러리를 가져옵니다
  2. driver = webdriver.Chrome(): Chrome 브라우저를 엽니다
  3. try:: 에러가 발생해도 브라우저를 닫을 수 있도록 try-finally를 사용합니다
  4. driver.get("https://www.google.com"): 구글 홈페이지로 이동합니다
  5. WebDriverWait(driver, 10).until(...): 검색창이 나타날 때까지 최대 10초 기다립니다
  6. By.NAME, "q": 구글 검색창은 name 속성이 "q"입니다. 개발자 도구로 확인할 수 있습니다!
  7. search_box.send_keys("Python Selenium 자동화"): 검색창에 검색어를 입력합니다
  8. Keys.RETURN: Enter 키를 누릅니다 (검색 실행)
  9. EC.presence_of_element_located((By.ID, "search")): 검색 결과 영역이 나타날 때까지 기다립니다
  10. driver.find_elements(By.CSS_SELECTOR, "h3"): 모든 제목(h3 태그)을 찾습니다
  11. enumerate(results[:5], 1): 처음 5개 결과만 가져와서 번호를 매깁니다
  12. finally: driver.quit(): 에러가 나도 브라우저를 반드시 닫습니다

실행 방법

  1. 위 코드를 google_search.py 파일로 저장합니다
  2. 터미널을 열고 파일이 있는 폴더로 이동합니다
  3. python google_search.py를 실행합니다
  4. 브라우저가 열리고 구글 검색이 자동으로 실행됩니다!

💡 수정해서 사용하기

  • 검색어 변경: search_box.send_keys("원하는 검색어") 부분을 수정하세요
  • 결과 개수 변경: results[:5]를 results[:10]으로 바꾸면 10개 결과를 가져옵니다
  • 다른 사이트: URL을 변경하면 다른 검색 사이트도 사용할 수 있습니다

로그인은 많은 웹사이트에서 자주 사용하는 기능입니다. 이 예제를 배우면 로그인 자동화를 만들 수 있습니다!

  1. 브라우저 열기
  2. 로그인 페이지 접속
  3. 사용자명 입력 필드 찾기
  4. 사용자명 입력
  5. 비밀번호 입력 필드 찾기

login_form.py 
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

driver = webdriver.Chrome()

try:
    # 로그인 페이지 열기 (예시)
    driver.get("https://example.com/login")
    
    # 사용자명 입력 필드 찾기
    username_field = WebDriverWait(driver, 10).until(
        EC.presence_of_element_located((By.ID, "username"))
    )
    username_field.clear()
    username_field.send_keys("your_username")
    
    # 비밀번호 입력 필드 찾기
    password_field = driver.find_element(By.ID, "password")
    password_field.clear()
    password_field.send_keys("your_password")
    
    # 로그인 버튼 클릭
    login_button = driver.find_element(By.CSS_SELECTOR, "button[type='submit']")
    login_button.click()
    
    # 로그인 성공 확인 (예: 대시보드 페이지로 이동했는지 확인)
    WebDriverWait(driver, 10).until(
        EC.url_contains("dashboard")  # URL에 "dashboard"가 포함되는지 확인
    )
    
    print("✅ 로그인 성공!")
    
except Exception as e:
    print(f"❌ 오류 발생: {e}")
    
finally:
    driver.quit()

코드 상세 설명

📖 각 부분 설명:
  1. driver.get("https://example.com/login"): 로그인 페이지로 이동합니다. 실제 사용할 웹사이트 주소로 변경하세요!
  2. WebDriverWait(...).until(EC.presence_of_element_located(...)): 사용자명 입력 필드가 나타날 때까지 기다립니다
  3. username_field.clear(): 입력 필드에 이미 있는 텍스트를 지웁니다 (혹시 모를 기존 내용 제거)
  4. username_field.send_keys("your_username"): 실제 사용자명으로 변경하세요!
  5. password_field = driver.find_element(By.ID, "password"): 비밀번호 필드를 찾습니다. 이미 페이지가 로드되었으므로 대기가 필요 없을 수 있습니다
  6. password_field.clear(): 비밀번호 필드도 지웁니다
  7. password_field.send_keys("your_password"): 실제 비밀번호로 변경하세요! (보안 주의!)
  8. By.CSS_SELECTOR, "button[type='submit']": type이 "submit"인 버튼을 찾습니다. 로그인 버튼의 실제 선택자로 변경해야 할 수 있습니다
  9. EC.url_contains("dashboard"): 로그인 성공 후 URL에 "dashboard"가 포함되는지 확인합니다. 실제 웹사이트에 맞게 변경하세요
  10. except Exception as e:: 에러가 발생하면 에러 메시지를 출력합니다

⚠️ 실제 사용 시 주의사항

  • 요소 선택자 변경: 실제 웹사이트의 요소 ID나 클래스를 개발자 도구로 확인하고 변경해야 합니다
  • 보안: 비밀번호를 코드에 직접 넣지 마세요! 환경 변수나 설정 파일을 사용하는 것을 권장합니다
  • 요소 찾기: 개발자 도구(F12)로 실제 로그인 폼의 요소를 확인하세요
  • 대기 시간: 느린 웹사이트는 대기 시간을 늘려야 할 수 있습니다

예제 3: 여러 페이지 크롤링

예시:

  • 쇼핑몰에서 모든 상품의 이름과 가격을 수집
  • 뉴스 사이트에서 모든 기사의 제목을 수집

작업 흐름

    driver.get("https://example.com/products")
    
    all_products = []  # 수집한 모든 상품을 저장할 리스트
    
    # 여러 페이지를 순회 (1페이지부터 3페이지까지)
    for page in range(1, 4):
        print(f"\n📄 {page}페이지 처리 중...")
        
        # 상품 목록 찾기 (여러 개의 상품 요소 찾기)
        products = WebDriverWait(driver, 10).until(
            EC.presence_of_all_elements_located((By.CLASS_NAME, "product-item"))
        )
        
        # 각 상품의 정보 추출
        for product in products:
            try:
                # 상품 이름 찾기
                name = product.find_element(By.CLASS_NAME, "product-name").text
                # 상품 가격 찾기
                price = product.find_element(By.CLASS_NAME, "product-price").text
                
                # 수집한 정보를 리스트에 추가
                all_products.append({
                    "name": name,
                    "price": price,
                    "page": page
                })
                
                print(f"  - {name}: {price}")
            except:
                # 에러가 발생하면 다음 상품으로 넘어감
                continue
        
        # 다음 페이지로 이동 (다음 버튼 클릭)
        try:
            next_button = driver.find_element(By.CSS_SELECTOR, ".pagination .next")
            if next_button.is_enabled():  # 버튼이 클릭 가능한지 확인
                next_button.click()
                time.sleep(2)  # 페이지 로드 대기
            else:
                break  # 다음 페이지가 없으면 종료
        except:
            break  # 다음 버튼을 찾을 수 없으면 종료
    
    print(f"\n✅ 총 {len(all_products)}개의 상품을 수집했습니다!")
    
finally:
    driver.quit()

코드 상세 설명

📖 각 부분 설명:
  1. all_products = []: 수집한 데이터를 저장할 빈 리스트(목록)를 만듭니다
  2. for page in range(1, 4):: 1부터 3까지 반복합니다 (1페이지, 2페이지, 3페이지)
  3. EC.presence_of_all_elements_located: 여러 개의 요소를 찾을 때 사용합니다 (find_elements와 유사)
  4. for product in products:: 찾은 각 상품에 대해 반복합니다
  5. product.find_element(...): 상품 요소 안에서 이름과 가격을 찾습니다 (상대적 요소 찾기)
  6. all_products.append({...}): 수집한 정보를 딕셔너리 형태로 리스트에 추가합니다
  7. next_button.is_enabled(): 버튼이 클릭 가능한지 확인합니다 (비활성화된 버튼은 클릭하지 않음)
  8. time.sleep(2): 페이지가 로드될 시간을 줍니다 (명시적 대기를 사용하는 것이 더 좋지만 간단한 예제로는 이것도 괜찮습니다)

💡 수정해서 사용하기

  • 페이지 수 변경: range(1, 4)를 range(1, 10)으로 바꾸면 9페이지까지 수집합니다
  • 데이터 저장: 수집한 데이터를 파일로 저장하려면 JSON이나 CSV 라이브러리를 사용하세요
  • 요소 선택자: 실제 웹사이트의 요소 클래스명이나 ID로 변경해야 합니다

3. 페이지 객체 모델 (Page Object Model, POM)

페이지 객체 모델이란? 각 웹페이지를 하나의 클래스로 만들어서 관리하는 디자인 패턴입니다.

코드가 더 깔끔하고 유지보수하기 쉬워집니다!

💡 왜 POM을 사용하나요?

문제: 모든 코드가 한 파일에 섞여 있으면:

  • 웹사이트 구조가 바뀌면 여러 곳을 수정해야 함
  • 코드가 길어져서 읽기 어려움
  • 같은 코드를 반복해서 작성

해결: 각 페이지를 클래스로 만들면:

  • 웹사이트가 바뀌어도 한 곳만 수정하면 됨
  • 코드가 깔끔하고 읽기 쉬움
  • 재사용 가능한 코드 작성
📖 POM의 핵심 개념:
  • 페이지 = 클래스: 각 웹페이지를 하나의 Python 클래스로 만듭니다
  • 요소 = 속성: 페이지의 요소들(버튼, 입력창 등)을 클래스의 속성으로 정의합니다
  • 동작 = 메서드: 페이지에서 할 수 있는 동작들을 클래스의 메서드(함수)로 만듭니다
페이지 객체 모델 구현 
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

# 페이지 객체 클래스 (로그인 페이지를 클래스로 만들기)
class LoginPage:
    def __init__(self, driver):
        self.driver = driver  # 브라우저 드라이버 저장
        self.wait = WebDriverWait(driver, 10)  # 대기 객체 저장
        
        # 요소 위치 정의 (나중에 쉽게 변경 가능)
        self.username_field = (By.ID, "username")
        self.password_field = (By.ID, "password")
        self.login_button = (By.CSS_SELECTOR, "button[type='submit']")
        self.error_message = (By.CLASS_NAME, "error")
    
    def open(self, url):
        """페이지 열기"""
        self.driver.get(url)
        return self  # 체이닝을 위해 self 반환
    
    def enter_username(self, username):
        """사용자명 입력"""
        field = self.wait.until(EC.presence_of_element_located(self.username_field))
        field.clear()
        field.send_keys(username)
        return self  # 체이닝을 위해 self 반환
    
    def enter_password(self, password):
        """비밀번호 입력"""
        field = self.driver.find_element(*self.password_field)
        field.clear()
        field.send_keys(password)
        return self  # 체이닝을 위해 self 반환
    
    def click_login(self):
        """로그인 버튼 클릭"""
        button = self.wait.until(EC.element_to_be_clickable(self.login_button))
        button.click()
        return self  # 체이닝을 위해 self 반환
    
    def get_error_message(self):
        """에러 메시지 가져오기"""
        try:
            error = self.driver.find_element(*self.error_message)
            return error.text
        except:
            return None  # 에러 메시지가 없으면 None 반환
    
    def login(self, username, password):
        """로그인 전체 프로세스 (편의 메서드)"""
        self.enter_username(username)
        self.enter_password(password)
        self.click_login()
        return self

# 사용 예제
driver = webdriver.Chrome()

try:
    # 방법 1: 단계별로 사용
    login_page = LoginPage(driver)
    login_page.open("https://example.com/login")
    login_page.login("user123", "password123")
    
    # 방법 2: 체이닝 방식 (더 깔끔함)
    login_page = LoginPage(driver)
    login_page.open("https://example.com/login") \
              .enter_username("user123") \
              .enter_password("password123") \
              .click_login()
    
finally:
    driver.quit()

POM 사용의 장점

✅ 유지보수 용이

웹사이트 요소가 바뀌어도 클래스 내부만 수정하면 됩니다

예: 버튼 ID가 바뀌면 self.login_button = (By.ID, "new-id")만 변경

✅ 코드 재사용

같은 페이지를 여러 테스트에서 재사용할 수 있습니다

예: login_page.login("user", "pass")를 여러 곳에서 사용

✅ 가독성 향상

코드가 읽기 쉽고 이해하기 쉬워집니다

예: login_page.enter_username("user")가 명확함

✅ 테스트와 코드 분리

테스트 로직과 페이지 조작을 분리할 수 있습니다

예: 테스트는 "무엇을" 테스트하는지만, POM은 "어떻게" 조작하는지만

4. 멀티 윈도우 및 탭 처리

윈도우/탭 관리 
from selenium import webdriver
from selenium.webdriver.common.by import By

driver = webdriver.Chrome()

try:
    driver.get("https://example.com")
    
    # 현재 윈도우 핸들 저장
    main_window = driver.current_window_handle
    print(f"메인 윈도우: {main_window}")
    
    # 새 탭 열기
    driver.execute_script("window.open('https://google.com', '_blank');")
    
    # 모든 윈도우 핸들 가져오기
    all_windows = driver.window_handles
    print(f"모든 윈도우: {all_windows}")
    
    # 새 탭으로 전환
    for window in all_windows:
        if window != main_window:
            driver.switch_to.window(window)
            print(f"새 탭 제목: {driver.title}")
            break
    
    # 메인 윈도우로 돌아가기
    driver.switch_to.window(main_window)
    
    # 새 윈도우 열기
    driver.execute_script("window.open('https://github.com', '_blank');")
    
    # 특정 윈도우로 전환
    for window in driver.window_handles:
        driver.switch_to.window(window)
        if "GitHub" in driver.title:
            print(f"GitHub 윈도우로 전환: {driver.title}")
            break
    
    # 윈도우 닫기

from selenium import webdriver
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from selenium.webdriver.common.by import By
from selenium.common.exceptions import TimeoutException

driver = webdriver.Chrome()

try:
    driver.get("https://example.com")
    
    # 커스텀 대기 조건 만들기
    class element_has_text:
        def __init__(self, locator, text):
            self.locator = locator
            self.text = text
        
        def __call__(self, driver):
            element = driver.find_element(*self.locator)
            if self.text in element.text:
                return element
            return False
    
    # 커스텀 대기 사용
    wait = WebDriverWait(driver, 10)
    element = wait.until(element_has_text((By.ID, "status"), "완료"))
    
    # 여러 조건 중 하나라도 만족할 때까지 대기
    from selenium.webdriver.support.wait import WebDriverWait
    
    wait = WebDriverWait(driver, 10)
    element = wait.until(
        lambda driver: driver.find_element(By.ID, "element1") or 
                      driver.find_element(By.ID, "element2")
    )
    
    # 타임아웃 예외 처리
    try:
        element = WebDriverWait(driver, 5).until(
            EC.presence_of_element_located((By.ID, "slow-element"))
        )
    except TimeoutException:
        print("요소를 찾을 수 없습니다. 타임아웃!")
    
    # 폴링 간격 조정 (기본 0.5초)
    wait = WebDriverWait(driver, timeout=10, poll_frequency=1)
    element = wait.until(EC.presence_of_element_located((By.ID, "element")))
    
finally:
    driver.quit()

🧪 Pytest 기초

Pytest란?

Pytest는 Python의 강력한 테스트 프레임워크입니다. Selenium 테스트를 더 체계적으로 작성하고 실행할 수 있게 해줍니다.

💡 테스트 프레임워크란?

테스트 프레임워크는 자동화 테스트를 쉽게 작성하고 실행할 수 있게 해주는 도구입니다.

비유: 레고 블록처럼 테스트를 쉽게 만들고 조립할 수 있는 도구상자와 같습니다!

✨ 주요 장점

  • 간단한 문법: def test_함수명():만 작성하면 됩니다
  • 자동 발견: test_로 시작하는 파일과 함수를 자동으로 찾아서 실행합니다
  • 상세한 보고: 실패한 테스트의 정확한 위치와 이유를 알려줍니다
  • 플러그인: HTML 리포트, 병렬 실행 등 다양한 기능을 추가할 수 있습니다
  • Fixture: 테스트 전후에 자동으로 실행되는 코드를 쉽게 만들 수 있습니다

Pytest 없이: 각 테스트를 하나씩 실행하고, 결과를 확인하기 어렵습니다

@pytest.hookimpl(tryfirst=True, hookwrapper=True)
def pytest_runtest_makereport(item, call):
    """테스트 결과 보고"""
    outcome = yield
    rep = outcome.get_result()
    
    if rep.when == "call" and rep.failed:
        # 실패한 테스트의 스크린샷 저장
        if "driver" in item.funcargs:
            driver = item.funcargs["driver"]
            screenshot_dir = "screenshots"
            os.makedirs(screenshot_dir, exist_ok=True)
            screenshot_path = os.path.join(
                screenshot_dir, 
                f"{item.name}_{datetime.now().strftime('%Y%m%d_%H%M%S')}.png"
            )
            driver.save_screenshot(screenshot_path)
            print(f"스크린샷 저장: {screenshot_path}")

자주 마주치는 문제들

❌ 문제 1: 요소를 찾을 수 없음 (NoSuchElementException)

원인: 페이지가 완전히 로드되기 전에 요소를 찾으려고 시도했거나, 요소의 위치가 잘못되었습니다.

해결 방법:

해결 코드 
# ❌ 나쁜 예: 즉시 요소 찾기
element = driver.find_element(By.ID, "my-element")

# ✅ 좋은 예: 명시적 대기 사용
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

wait = WebDriverWait(driver, 10)
element = wait.until(EC.presence_of_element_located((By.ID, "my-element")))

❌ 문제 2: 클릭이 작동하지 않음

원인: 요소가 아직 클릭 가능한 상태가 아니거나, 다른 요소에 가려져 있습니다.

해결 방법:

해결 코드 
# ✅ 요소가 클릭 가능할 때까지 대기
wait = WebDriverWait(driver, 10)
button = wait.until(EC.element_to_be_clickable((By.ID, "my-button")))
button.click()

# 또는 JavaScript로 클릭 (마지막 수단)
button = driver.find_element(By.ID, "my-button")
driver.execute_script("arguments[0].click();", button)

❌ 문제 3: 브라우저가 너무 빨리 닫힘

원인: 스크립트가 너무 빨리 실행되어 결과를 볼 시간이 없습니다.

해결 방법:

해결 코드 
# ✅ 마지막에 입력 대기 또는 시간 지연
import time

# ... 작업 수행 ...

# 사용자 입력 대기 (Enter 키 누를 때까지)
input("작업이 완료되었습니다. Enter 키를 눌러 브라우저를 닫으세요...")
driver.quit()

# 또는 일정 시간 대기
time.sleep(5)  # 5초 대기
driver.quit()

베스트 프랙티스

✅ 1. 항상 try-finally 사용

예외가 발생해도 브라우저가 확실히 닫히도록 합니다.

✅ 2. 명시적 대기 사용

time.sleep() 대신 WebDriverWait를 사용하면 더 안정적이고 빠릅니다.

✅ 3. 적절한 Locator 선택

ID > CSS Selector > XPath 순으로 우선순위를 두세요. ID가 가장 빠르고 안정적입니다.

✅ 4. 페이지 객체 모델 (POM) 패턴 사용

큰 프로젝트에서는 각 페이지를 클래스로 만들어 관리하면 유지보수가 쉬워집니다.
